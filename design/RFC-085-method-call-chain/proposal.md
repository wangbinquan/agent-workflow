# RFC-085 — 方法调用链 / 时序图（Method Call Chain & Sequence Diagram）

状态：**Draft**（待用户批准进入实现）

> 编号说明：用户口头说"起草 RFC-084",但 `RFC-084-conformance-auditor` 已被并行协作者占用,本 RFC 顺延为 **RFC-085**。

## 背景

RFC-083 给出了结构化 diff 的**类协作图**(类/方法卡片 + 继承/引用/调用边 + 方法级高亮)。它回答的是"**谁和谁有关系**"(类与类、方法与方法的静态关联),是一张**无向/双向**的关系网。

但用户在 review 一个 agent 改动时,经常想问的是另一类问题——**"这个方法点下去,后续会一路调到哪些方法?按什么顺序?"**:

- 入口方法 `GameEngine.tick()` → 调 `MovementHandler.move()` → 调 `CollisionDetector.check()` → 调 `GameContext.getSnake()` …
- 这是一条**有方向、有顺序、可递归展开**的**调用链**;最自然的呈现是**时序图(sequence diagram)**:参与方(类/对象)作为 lifeline,调用作为自上而下的有序消息。

RFC-083 的数据**不足以**直接做这件事:`impact` 是**反向**的(「谁调了我」)且稀疏;按名匹配只产出"调用方↔被调方"的**无序对**。本 RFC 增加**正向、带顺序**的调用提取 + 调用链遍历 + 时序图可视化。

## 目标

1. 在结构视图里,**点击一个方法**(树视图的成员行 / 关系图的方法行),弹出该方法的**后续调用链**。
2. 调用链**可递归展开**(逐层往下:M 调了谁、它们又调了谁),带**环检测**与**深度上限**。
3. 提供**时序图视图**:lifeline = 参与的类/接收者类型,消息 = 调用,**自上而下按源码顺序**,支持嵌套/激活条。
4. **静态、确定性、可复现优先**(与 RFC-083 同政策):调用提取来自 tree-sitter / SCIP,无 LLM。
5. **诚实的精度边界**:能解析的调用精确呈现,解析不了的(动态分派、链式、泛型、lambda、跨二方库)显式标"未解析",**绝不臆造**调用关系。
6. **复用 RFC-083 既有产物**:符号集(`SymbolNode`)、成员范围、深度模式 SCIP —— 不另起一套提取管线。

## 非目标(本 RFC)

- **不做运行时 / 动态调用追踪**:纯静态,不插桩、不跑代码。
- **不追求 100% 调用图完整性**:完整度受限于提取器(基线 tree-sitter / 深度 SCIP 覆盖),这是已知边界、会显式标注,不是 bug。
- **不做跨语言调用**(FFI / RPC / 反射字符串):超出静态可达范围。
- **不替代 RFC-083 关系图**:时序图是**补充的第二视图**,从关系图/树**进入**,不改既有图。
- **不引入 LLM 摘要**:留作后置可选增强。

## 用户故事

1. **追一条主流程**:我在结构树点 `GameEngine.tick()`,右侧弹出时序图:`tick → MovementHandler.move → CollisionDetector.check → GameContext.getSnake`,一眼看清这次 tick 的执行骨架。
2. **逐层展开**:调用链默认展开 2 层;我点某个被调方法旁的"▸"再往下钻,看它内部还调了什么,带深度上限防爆炸。
3. **顺序可信**:时序图里消息的上下顺序 = 源码里调用语句的先后(不是字母序、不是随机),所以能当"执行骨架"读。
4. **精度透明**:`renderer.draw(g)` 这种能解析到 `SnakeRenderer.draw` 的精确连;`factory.create(cfg).run()` 这种链式/无法定位接收者类型的,标成"未解析调用"灰显,不假装连到某个类。
5. **从关系图进入**:我在 RFC-083 关系图里点中某方法行,有个"看调用链"入口直接切到本视图,上下文连续。

## 验收标准(产品层)

- 给定一对 before/after(或当前 worktree)+ 一个入口方法,**正向**列出它**直接调用**的、能静态解析到具体方法的调用集合(阶段 1)。
- 调用链能**递归展开 ≥3 层**,有**环检测**(A→B→A 不死循环)与**深度/节点上限**(防超大图),命中上限时显式标"已截断"。
- 时序图:消息**按源码顺序**自上而下;参与类作为 lifeline;至少覆盖 Java + TS/JS(其余语言 best-effort)。
- **无法解析**的调用以独立样式呈现(灰显 / "未解析"),不并入精确链、不臆造目标。
- 深度模式(SCIP)可用时调用解析更准;不可用自动回退基线启发式,视图不崩、显式标"基线精度"。
- 全绿门槛:`bun run typecheck && bun run test && bun run format:check`(+ 单二进制 smoke,若动到后端/shared)。
