# RFC-190 · 首页能力门户改版：动态管线 hero + 能力卡片矩阵 + 任务动态列

状态：Draft（待 Codex 设计门 + 用户批准）
日期：2026-07-15
来源：用户需求「优化首页的显示，要有平台特点，要直观体现平台能力」；四项设计取向已与用户问答对齐（形态=能力门户 / hero=动态管线画布 / 数据=新增聚合端点 / 范围=连 Onboarding 一起翻新）。

## 1、背景与问题

现首页是 RFC-032 落的任务仪表盘：问候语 + 运行时状态行 + 「等你处理 / 运行中 / 最近完成」三个列表区。它解决了"回来看任务"的运营诉求，但有三个问题：

1. **没有平台特点**。本平台的差异化能力——把「编码 → 审计 → 修复」这类多代理流水线交给确定性引擎编排（git 快照 → 进程级隔离执行 → 扇出 → 聚合）——在首页完全不可见；首页视觉上是一个通用任务列表，与任意工单系统无异。
2. **不体现能力边界**。代理 / 技能 / MCP / 插件 / 工作流 / 工作组 / 定时任务 / 仓库 / 记忆这些资源能力全部埋在侧栏导航里，首页没有一个「平台能有什么、我已经有什么」的地图；新用户（以及空闲期回访的老用户）打开首页得不到任何能力感知。
3. **空态即白板**。无任务时三个区块全是空态文案，整页几乎全白，是体验最差的一屏。

## 2、目标

- **G1 能力门户形态**：首页主体改为**能力卡片矩阵**——代理（含技能 / MCP / 插件子计数）、工作流、工作组、记忆、定时任务、仓库六张卡，每张含实时计数 + 一句能力描述 + 点击跳转对应列表页，构成平台能力地图。
- **G2 动态管线 hero**：顶部 hero 用纯 SVG/CSS 动画渲染一幅迷你工作流管线（验收修订 v3：`(输入) → GIT wrapper[编码] —git_diff→ 审计×3 扇出 → 修复 → (输出)`——与真实画布同构的业务流：快照/聚合是框架机制不画成节点，git wrapper 画为虚线容器、聚合由多进程节点内建），连线上有流动光点、节点微呼吸——把平台核心抽象「record → run → fan-out → aggregate」直观画在首页上。保留问候语、运行时状态行与「启动任务」主行动。
- **G3 任务动态单列**：现有三个任务区合并为一个「任务动态」区块（等你处理优先置顶、运行中次之、最近完成殿后），压缩纵向占位，把首屏让给能力表达；行为与数据口径不变。
- **G4 聚合取数**：后端新增 `GET /api/overview`，一次请求返回上述各资源**当前用户可见**的计数与任务 7 天窗口统计（运行中 / 等待人 / 7d 完成 / 7d 失败），口径与各列表页一致（RFC-099 ACL 过滤）。
- **G5 Onboarding 同步翻新**：首跑引导页套用同款管线 hero 与能力介绍（无计数），首跑第一屏即传达平台能力，与常规首页视觉连续。
- **G6 设计体系一致**：全部复用既有公共组件 / class 体系；light + dark 双主题；动画尊重 `prefers-reduced-motion`；i18n 双语齐备。

## 3、非目标

- 不做首页布局个性化 / 卡片增删自定义。
- 不改各资源列表页与任务详情页。
- 不引入任何图表库 / 动画库（hero 为手写 SVG + CSS）。
- 不做任务/资源数据模型变更、无 DB migration。
- 不上 WebSocket 推送；沿用现有轮询节奏（overview 轮询间隔见 design.md）。
- 不做营销文案页；能力描述一句话为限，服务导航而非宣传。

## 4、用户故事

- **US1 首跑用户**：首次启动打开首页，看到管线动画与平台能力介绍，理解「这是一个把多代理流水线可视化编排的平台」，并被引导创建第一个代理 / 工作流。
- **US2 日常用户**：回到首页一眼看到：等我处理的评审/反问（置顶）、在跑的任务、以及我可用的资源规模（6 张卡计数）；点卡片直达对应列表页。
- **US3 多用户 ACL**：普通用户 B 打开首页，卡片计数只统计 B 可见的资源（owner / public / 授权），与 B 打开各列表页看到的行数一致；任务计数只含 B 为成员的任务。
- **US4 无障碍 / 主题**：系统开启「减少动态效果」时管线动画静止为静态图；暗色主题下 hero 与卡片对比度正常。

## 5、验收标准

- [ ] 首页（非首跑）呈现：hero（问候 + 运行时状态 + 管线动画 + 启动任务）→ 能力卡片矩阵（6 卡实时计数）→ 任务动态单列（等你处理 / 运行中 / 最近完成三组保序合并）。
- [ ] `GET /api/overview` 返回六类资源计数 + 代理子计数（技能/MCP/插件）+ 任务统计；对同一用户，各计数与对应列表接口行数一致（含 ACL 隔离用例：陌生用户看不到他人 private 资源）。
- [ ] 卡片点击跳转：代理→/agents、工作流→/workflows、工作组→/workgroups、记忆→/memory、定时→/scheduled、仓库→/repos；任务动态各组保留原跳转。
- [ ] Onboarding 首跑页含同款管线 hero 与能力介绍，原创建引导 CTA 保留可用。
- [ ] 管线动画零第三方依赖；`prefers-reduced-motion: reduce` 下无移动动画；light/dark 均通过视觉自查（与 /agents、/workflows 等核心页并排对齐）。
- [ ] i18n：zh-CN / en-US 全部新增 key 齐备，无硬编码文案。
- [ ] 测试：后端 overview 路由测试（口径 oracle + ACL 隔离 + 权限真值表 + 固定时钟 7d 边界）+ API contract registry 登记；前端 vitest 覆盖新组件与合并后任务动态；既有首页测试锁按新结构迁移不减断言强度；e2e 视觉/a11y 拆「空库 Onboarding」与「种子后 Homepage」两组基线；`bun run typecheck && bun run lint && bun run test && bun run format:check` 全绿。
